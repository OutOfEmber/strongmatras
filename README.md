2.Перечислите и опишите все допустимые значения свойства transform.
none: (по умолчанию) Отсутствие трансформации.
Перемещение:
translateX(tx) - перемещает элемент по оси X.
translateY(ty) - перемещает элемент по оси Y.
translate(tx, ty) - сокращенная запись для X и Y.
translateZ(tz) - перемещает элемент по оси Z (в 3D-пространстве).
translate3d(tx, ty, tz) - перемещает элемент по всем трем осям.
Масштабирование:
scaleX(sx) - масштабирует элемент по оси X.
scaleY(sy) - масштабирует элемент по оси Y.
scale(sx, sy) - масштабирует элемент по осям X и Y. Если один аргумент, он используется для обеих осей.
scaleZ(sz) - масштабирует элемент по оси Z.
scale3d(sx, sy, sz) - масштабирует элемент по всем трем осям.
Поворот:
rotate(angle) - поворачивает элемент в 2D-пространстве на заданный угол (например, 45deg).
rotateX(angle) - поворачивает элемент вокруг оси X.
rotateY(angle) - поворачивает элемент вокруг оси Y.
rotateZ(angle) - поворачивает элемент вокруг оси Z (аналогично rotate).
rotate3d(x, y, z, angle) - задает произвольный вектор [x,y,z], вокруг которого происходит поворот.
Наклон:
skewX(ax) - наклоняет элемент по оси X на заданный угол.
skewY(ay) - наклоняет элемент по оси Y на заданный угол.
skew(ax, ay) - наклоняет элемент по осям X и Y.
Перспектива:
perspective(n) - задает глубину для 3D-трансформации дочерних элементов. Обычно лучше использовать отдельное свойство perspective.
пример:
.element {
    transform: translate(50px, 10px) rotate(15deg) scale(1.1);
}

3.Какие единицы измерения в CSS3 вы знаете? Для чего используются те или иные единицы измерения? Приведите примеры.
Абсолютные: Не зависят от контекста. (Использовать в вебе реже)
px (пиксели) - Базовая единица для экранов.
cm, mm, in (сантиметры, миллиметры, дюймы) - Для печати.
Относительные:
Относительно шрифта:
em - Относится к размеру шрифта родительского элемента. 1.5em = 150% от размера шрифта родителя.
rem - Относится к размеру шрифта корневого элемента (<html>). Более предсказуем, чем em.
ex, ch - Относится к высоте символа "x" и ширине символа "0" текущего шрифта.
Относительно окна просмотра (Viewport):
vw - 1% от ширины окна браузера.
vh - 1% от высоты окна браузера.
vmin - 1% от меньшей стороны окна (ширины или высоты).
vmax - 1% от большей стороны окна.
% - Относится к значению того же свойства у родительского элемента. Для width: 50% элемент займет половину ширины родителя.

5.Опишите механизм наследования и каскадирования стилей.
Наследование: Некоторые CSS-свойства (например, color, font-family, font-size) автоматически передаются от родительского элемента к дочерним. Это позволяет задавать стили для всего документа, установив их для элемента <body>.
Управление: С помощью значения inherit можно явно заставить элемент унаследовать значение свойства у родителя, даже если свойство по умолчанию не наследуется.
Каскадирование: Это алгоритм, который определяет, какие именно CSS-правила будут применены к элементу, когда на него влияют несколько конфликтующих правил. Он работает по трем факторам (в порядке важности):
Важность (!important): Объявление с !important имеет наивысший приоритет.
Специфичность: Вес селектора. Рассчитывается как (A, B, C, D):
A: Инлайн-стили (style="...").
B: Количество ID селекторов (#header).
C: Количество классов, атрибутов и псевдоклассов (.class, :hover).
D: Количество тегов и псевдоэлементов (div, ::before).
Пример: #nav .item:hover имеет специфичность (0, 1, 2, 0).
Порядок исходного кода: Если два правила имеют одинаковую важность и специфичность, то последнее объявленное в коде правило будет применено.

6.Опишите блочную модель расположения элементов.
Содержимое (Content): Область, где отображается текст, изображения и другой контент. Размеры задаются свойствами width и height.
Внутренние отступы (Padding): Прозрачная область вокруг содержимого, отделяющая его от границы. Задается свойством padding.
Граница (Border): Линия, окружающая внутренние отступы и содержимое. Задается свойством border.
Внешние отступы (Margin): Прозрачная область вокруг границы, которая отталкивает другие элементы. Задается свойством margin.
Важно: По умолчанию свойства width и height задают размер только содержимого. Общая ширина/высота элемента рассчитывается как:
width/height + padding + border + margin.
Свойство box-sizing: Позволяет изменить эту логику.
content-box (по умолчанию): Классическая модель, описанная выше.
border-box: Свойства width и height включают в себя содержимое, внутренние отступы и границу. Внешние отступы (margin) не входят.

7.Какие вы знаете строчные и блочные элементы? В чем различие? Как перейти от строчного элемента к блочному?
Блочные элементы (display: block):
Примеры: <div>, <h1>-<h6>, <p>, <section>, <ul>, <li>.
Особенности:
Занимают всю доступную ширину родителя.
Начинаются с новой строки.
Можно задавать width, height, все отступы (margin, padding).
Строчные элементы (display: inline):
Примеры: <span>, <a>, <strong>, <em>, <img>.
Особенности:
Занимают только ширину своего содержимого.
Располагаются на одной строке с другими строчными элементами.
Нельзя задавать width, height, вертикальные отступы (margin-top, margin-bottom).
Переход: С помощью CSS-свойства display можно изменить тип элемента.
display: block; - сделать строчный элемент блочным.
display: inline; - сделать блочный элемент строчным.
display: inline-block; - гибрид: элемент ведет себя как строчный (располагается в строке), но ему можно задавать ширину, высоту и отступы, как блочному.

8.Что такое медиа-запрос, как он работает и для чего применяется?
Медиа-запрос (Media Query) — это технология в CSS3, которая позволяет применять разные CSS-стили в зависимости от характеристик устройства или окна браузера.
Как работает: Браузер проверяет условия, указанные в медиа-запросе (например, ширина экрана). Если условие истинно (true), то блок CSS-стилей внутри этого запроса применяется. Если ложно (false) — игнорируется.
Для чего применяется: Основа адаптивного веб-дизайна (RWD). Позволяет создавать сайты, которые оптимально отображаются на устройствах с разными размерами экранов (телефоны, планшеты, десктопы).

9.Опишите структуру медиа-запроса.
структура состоит из:
Ключевого слова: @media или @import.
Типа носителя (Media Type): all, screen, print и т.д. (необязателен, по умолчанию all).
Логического оператора: and, not, only, , (запятая как "или").
Медиа-функции (Media Feature): Условия в круглых скобках, например (min-width: 768px) или (orientation: landscape).

10.Какие типы носителя для медиа-запросов вы знаете? Перечислите и опишите известные типы.
all - Подходит для всех устройств. (По умолчанию)
screen - Предназначен в первую очередь для экранов (мониторы, планшеты, телефоны).
print - Предназначен для печатающих устройств и режима предварительного просмотра для печати.
speech - Предназначен для синтезаторов речи, которые зачитывают страницу вслух.

11.С помощью каких логических операторов можно комбинировать медиа-запросы? Приведите примеры.
and - Логическое "И". Объединяет несколько условий. Все должны быть истинны.
/* Для экранов в альбомной ориентации и шириной не менее 1024px */
@media screen and (orientation: landscape) and (min-width: 1024px) { ... }
, (запятая) - Логическое "ИЛИ". Разделяет несколько запросов, стили применятся, если истинен любой из них.
/* Для экранов с максимальной шириной 600px ИЛИ в портретной ориентации */
@media (max-width: 600px), (orientation: portrait) { ... }
not - Логическое "НЕ". Отрицает весь медиа-запрос. Применяет стили, если условие ложно.
/* Применяются ко всем устройствам, кроме принтеров */
@media not print { ... }
only - Скрывает таблицу стилей от старых браузеров, которые не поддерживают медиа-запросы. Они увидят стили, только если запрос истинен.
@media only screen and (min-width: 768px) { ... }
